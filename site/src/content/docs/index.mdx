**Status:** v0.2.0 on PyPI. Install with `uv tool install quartobot`.
This page is itself rendered through `quartobot resolve` — see
[first-manuscript](./first-manuscript/) for the dogfood loop.

quartobot is a small set of composable pieces. Pick the layer that
matches what you're trying to do — they work alone or stacked.

<div class="feature-band">
  <div class="feature-band__body">
    <p class="feature-band__audience">For: shell-tool agents · jq users · one-shot lookups</p>
    <h2 class="feature-band__title">1. Citation resolution</h2>
    <p>
      Atomic capability. Hand quartobot a persistent-identifier key —
      `doi:`, `pmid:`, `arxiv:`, `isbn:`, `url:`, `wikidata:`, `pmc:`
      — and get back CSL JSON. The resolver is
      `manubot.cite.citekey_to_csl_item` underneath, wrapped with a
      stdout mode so it pipes cleanly through `jq` or any agent's
      shell tool.
    </p>
    <p class="feature-band__cta">
      <a href="./resolve-single-citation/">How to resolve a single citation →</a>
    </p>
  </div>
  <div class="feature-band__code">

```bash
quartobot resolve --output - doi:10.21105/joss.01686 \
  | jq '.[0].title'
# → "Welcome to the Tidyverse"
```

  </div>
</div>

<div class="feature-band feature-band--reverse">
  <div class="feature-band__body">
    <p class="feature-band__audience">For: Quarto authors · manuscript, book, or website projects</p>
    <h2 class="feature-band__title">2. Pre-render hook for Quarto</h2>
    <p>
      One line in `_quarto.yml` makes `quartobot resolve` run before
      pandoc on every render. Authors write `@doi:…` in their prose;
      the hook fetches each key, writes a CSL JSON bibliography to
      `references.json`, and pandoc-citeproc formats the rendered
      output. Network only on first render; cached after.
    </p>
    <p class="feature-band__cta">
      <a href="./first-manuscript/">First manuscript in 15 minutes →</a>
    </p>
  </div>
  <div class="feature-band__code">

```yaml
# _quarto.yml
project:
  pre-render: >-
    quartobot resolve --from-scan .
    --output references.json
    --id-mode citation-key

bibliography:
  - references.bib
  - references.json
```

  </div>
</div>

<div class="feature-band">
  <div class="feature-band__body">
    <p class="feature-band__audience">For: drafting in Claude Desktop · Codex · Gemini · custom agent harnesses</p>
    <h2 class="feature-band__title">3. Citation grounding for agents</h2>
    <p>
      The same resolver, reached two ways. `quartobot mcp` starts a
      stdio MCP server with three read-only tools (`resolve_citation`,
      `scan_project`, `validate_project`) for native MCP clients.
      Agents without MCP shell out to `quartobot resolve --output -`
      and pipe through `jq`. Either way, every cite the agent
      proposes can be grounded against the registrar before it lands
      in the draft.
    </p>
    <p class="feature-band__cta">
      <a href="./mcp-claude-desktop/">MCP + Claude Desktop →</a>
      &nbsp;·&nbsp;
      <a href="./shell-tool-agent/">Shell-tool agents →</a>
    </p>
  </div>
  <div class="feature-band__code">

```json
// ~/Library/Application Support/Claude/
//   claude_desktop_config.json
{
  "mcpServers": {
    "quartobot": {
      "command": "quartobot",
      "args": ["mcp"]
    }
  }
}
```

    <p class="feature-band__audience">For: the full manuscript-as-software pattern · per-commit permalinks · PR previews</p>
    <h2 class="feature-band__title">4. GitHub automation</h2>
    <p>
      `quartobot use github-ci` scaffolds an opt-in render workflow:
      `.github/workflows/render.yml` (calls the upstream reusable
      workflow), the version-banner Quarto include, and a PR-preview
      cleanup workflow. Every commit gets an immutable `/v/<sha>/`
      permalink on GitHub Pages; every PR gets a sticky preview
      comment. Skip this layer if you already have CI or use
      `quarto publish`.
    </p>
    <p class="feature-band__cta">
      <a href="./first-manuscript/#7-add-the-publish-on-every-commit-ci">First manuscript, step 7 →</a>
    </p>
  </div>

```bash
quartobot use github-ci
# scaffolds:
#   .github/workflows/render.yml
#   .github/workflows/pr-closed.yml
#   _version-banner.html
# then prints a YAML snippet to paste
# into _quarto.yml for the banner include
```

## Modular by design

Same `manubot.cite.citekey_to_csl_item` call underneath all four
layers. No parallel implementations, no fork. Adopt one, two, three,
or all four — they compose, they don't require each other.
